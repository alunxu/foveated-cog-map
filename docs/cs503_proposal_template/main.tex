% ---------- Abstract ----------
\begin{abstract}
Recurrent navigation agents develop spatial maps in their LSTM hidden states even when their only input is GPS and compass~\cite{wijmans2023emergence}. This result, and the broader probing literature on navigation agents, was obtained under either uniform vision or no vision, leaving the middle of that range untested. Real biological vision is foveated: it is sharp at the center of gaze and degraded toward the periphery, so the agent has to decide what to remember about regions it could not see clearly. We ask whether a learned agent placed in this setting develops the same kind of cognitive map as a uniform-vision agent, or a different one that compensates for the uneven input. We train four PointGoal navigators on Gibson with DD-PPO---blind, uniform RGB, foveated RGB with a learned gaze head, and a matched-compute low-resolution control---and probe their hidden states for occupancy, position, and per-location perceptual confidence. The four-condition design lets us attribute any observed difference in memory content to sensor structure rather than task demands.
\end{abstract}


% ---------- Introduction ----------
\section{Introduction}

The cognitive map hypothesis~\cite{tolman1948cognitive}---that animals build internal spatial representations to support flexible navigation---has guided spatial cognition research for over seventy years. Wijmans et al.~\cite{wijmans2023emergence} recently gave it a computational handle: reinforcement learning agents trained on PointGoal navigation develop map-like representations in their recurrent hidden states, detectable by linear probes that decode occupancy grids, agent position, and collision predictions. The same paper showed that this occurs even in \emph{blind} agents receiving only GPS and compass---vision is not required for spatial memory to emerge.

This finding mapped one endpoint of the perceptual spectrum. The other endpoint---uniform high-resolution RGB---is what most embodied AI training assumes. Real biological vision is neither: humans see sharply only within the central $\sim$2\textdegree{} of gaze (the fovea), and resolution falls off rapidly with eccentricity. The agent therefore cannot simply ``see and remember''; it must choose where to look, and the periphery feeds memory with information that is uncertain by construction. We do not know whether a learned agent placed in this regime develops the same kind of cognitive map as a uniform-vision agent, or something qualitatively different that compensates for the uneven input.

\subsection{What is the problem?}

Existing probing studies of navigation agents~\cite{wijmans2023emergence,dwivedi2022isee} characterize \emph{what} cognitive maps emerge under uniform or absent vision; none vary the spatial quality of the visual input. As a result, we cannot tell whether the spatial concepts found in those hidden states are properties of the navigation task itself or artifacts of receiving every part of the scene at the same resolution. We ask: \textbf{when a navigation agent sees the world through a foveated sensor, what does it choose to remember, and does its memory reflect a strategy for managing perceptual uncertainty?}

\subsection{Why does this matter?}

\begin{itemize}
    \item \textbf{It distinguishes two theories of what cognitive maps are}\quad A learned cognitive map could be a \emph{world model}---a photo album that stores ``a chair is at $X$''---or a \emph{belief state}---an annotated notebook that stores ``a chair is probably at $X$, but I only saw it in my periphery.'' Existing probing work~\cite{wijmans2023emergence,dwivedi2022isee} cannot tell them apart: under uniform vision, confidence is uniformly high everywhere, so both accounts predict identical probe outputs. Foveation gives the agent regions it actually saw poorly. The two accounts then diverge---a world model's accuracy should drop with eccentricity, while a belief state's should stay flat or rise---and our H1/H2 tests target exactly this divergence.
    \item \textbf{It gives a concrete answer to ``what should a memory store?''}\quad Recurrent policies for partial observability (LSTMs, Transformer memories) currently store whatever the loss happens to push them toward, with no principled rule. If foveated agents develop uncertainty-driven storage (H1), that translates into a concrete inductive bias---weight memory updates by per-region observation quality---portable into any memory module for POMDP-style tasks.
    \item \textbf{It reframes learned gaze in terms of memory, not performance}\quad Active-vision RL~\cite{lv2023sugarl} shows learned gaze improves task return but treats gaze as a performance trick. Our H3 test asks whether the agent looks where its memory is weakest, turning an empirical trick into a mechanistic account.
\end{itemize}


% ---------- Method ----------
\section{Method and Deliveries}

\subsection{Experimental design}

\textbf{Simulator and dataset}\quad We train all agents in Habitat~\cite{savva2019habitat}, a high-throughput 3D simulator that renders first-person RGB at thousands of frames per second, on the Gibson dataset~\cite{xia2018gibson}, which provides 492 photorealistic 3D reconstructions of real buildings. The task is PointGoal navigation: the agent starts at a random pose and must reach a target specified as a displacement vector (distance and heading to goal) using discrete actions (\textsc{forward}, \textsc{turn-left}, \textsc{turn-right}, \textsc{stop}). We hold out Matterport3D (90 scenes) as a generalization test set at evaluation time.

\textbf{Four agents}\quad All four agents share an identical LSTM backbone (3 layers, 512-d) and differ only in the visual input fed to it. Any observed difference in memory content can then be attributed to sensor structure rather than backbone or task:

\noindent\textbf{Blind:} GPS+compass only (4D vector). Replicates~\cite{wijmans2023emergence}.\\
\textbf{Uniform:} Full-resolution RGB (256$\times$256) via ResNet18 encoder.\\
\textbf{Foveated:} Same RGB but with eccentricity-dependent Gaussian blur. A gaze decoder MLP ($512 \rightarrow 64 \rightarrow 2$, sigmoid) outputs gaze position from the previous LSTM hidden state. Gradients flow end-to-end through the foveation transform.\\
\textbf{Matched-compute:} Uniform low-resolution RGB (64$\times$64), with roughly the same total pixel budget as the foveated agent actually receives. Foveated vs.\ matched isolates whether any foveation effect comes from the fovea+periphery \emph{structure} or simply from having less total input.

\subsection{Hypotheses and validation}

We organize the analysis around three hypotheses.

\noindent\textbf{H1 (Uncertainty-driven memory)}\quad the foveated agent's memory is \emph{more} accurate for regions seen only peripherally than for regions seen at the fovea---memory compensates where perception is weakest. We test this by stratifying probe accuracy by observation eccentricity (Fig.~\ref{fig:setup}).

\noindent\textbf{H2 (Confidence-annotated maps)}\quad the hidden state encodes per-location observation quality, not just layout. We train a probe to decode the agent's cumulative input quality from gaze history.

\noindent\textbf{H3 (Gaze-memory coupling)}\quad gaze targets high-uncertainty regions, approximating information-gain-maximizing fixation~\cite{radulescu2022eyemovements}. We measure the correlation between gaze direction and decoded memory uncertainty against a Bayesian ideal observer.

\noindent\textbf{Metrics}\quad navigation success and SPL for task competence; probe accuracy (AUC, $R^2$) for H1/H2; CKA~\cite{kornblith2019cka} and MDL probe complexity~\cite{voita2020mdl} for cross-condition representation comparison.

% ---------- Related Work ----------
\section{Related Work}

\textbf{Cognitive maps in RL agents}\quad Wijmans et al.~\cite{wijmans2023emergence} showed that spatial maps emerge in blind agents' LSTM states. Banino et al.~\cite{banino2018grid} found grid-cell-like representations in path-integrating networks, and Gornet \& Thomson~\cite{gornet2024cognitive} extended this using visual predictive coding. Dwivedi et al.~\cite{dwivedi2022isee} probed PointNav agents for spatial concepts (reachability, progress, target visibility) and identified which neurons encode them. These studies all use uniform or absent perception---none vary the spatial quality of visual input.

\textbf{Foveated vision in deep learning}\quad Mnih et al.~\cite{mnih2014attention} introduced recurrent attention models that select image regions sequentially via RL, an early instance of learned gaze control in deep networks. More recently, Pourrahimi \& Bashivan~\cite{pourrahimi2025foveated} trained a foveated model on visual search and probed for neuroscience-predicted features, and Lv et al.~\cite{lv2023sugarl} jointly learned gaze and motor policies for RL under limited observability. These works study either static scenes or task performance---not how foveation changes what a navigation agent's memory encodes over time.

\textbf{Neuroscience motivation}\quad Wirth et al.~\cite{wirth2017gaze} showed that primate hippocampal spatial representations depend on what the animal foveates, not just where it is: foveation enhances landmark and path selectivity. Atanov et al.~\cite{atanov2024photoreceptor} showed computationally that sensor layout changes downstream task performance. Together these point to a hypothesis---input structure shapes spatial representations---that has not been tested in learned navigation agents.

\textbf{What our work adds}\quad These lines are individually mature but disconnected: probing studies use uniform vision, foveated-vision work does not probe spatial memory, and the foveation--hippocampus link has been reported in animals rather than learned agents. We connect them by varying only the spatial structure of the visual input across four matched agents and asking what changes in their hidden states.

% ---------- Discussion ----------
\section{Discussion}

\subsection{Implications and contribution}

If H1 holds, the foveated agent's memory tracks what perception \emph{cannot} currently provide rather than what it can---an uncertainty-compensation pattern that would offer a candidate answer to the otherwise underspecified question of what a recurrent memory should store under partial observability. If H2 holds, the agent represents not only spatial layout but also its own observation quality at each location, which is closer to a Bayesian belief state than to a plain map. If H3 holds, the learned gaze policy lines up with information-gain-maximizing fixation, offering computational support for rational-analysis accounts of eye movements~\cite{radulescu2022eyemovements}.

A null result would also be informative: if foveated and uniform agents end up with similar probe accuracies, it would suggest that cognitive map content is shaped primarily by the task rather than the sensor, and that the foveal--hippocampal link reported in primates~\cite{wirth2017gaze} does not straightforwardly transfer to learned agents.

\noindent\textbf{Scope of design choices}\quad We pick the simplest differentiable foveation (multi-scale Gaussian blur), the simplest end-to-end gaze (deterministic MLP from the LSTM hidden state), and the LSTM backbone from Wijmans et al.\ across all four conditions, so any observed effect is attributable to spatial non-uniformity itself rather than to architectural choices elsewhere. Richer alternatives---log-polar transforms, stochastic gaze, Scene Memory Transformer architectures---are well-motivated follow-ups once the baseline is in.

\subsection{Feasibility \& risk assessment}
\begin{itemize}
    \item \textbf{Data:} Gibson (492 scenes, 13\,GB) and Matterport3D (90 scenes, 20\,GB) are downloaded and verified on SCITAS. PointNav episodes are publicly available.
    \item \textbf{Compute:} Training on EPFL's Izar cluster (V100 GPUs). Blind agent: $\sim$6 days on 2 GPUs; sighted agents: $\sim$4 days each. All jobs fit within the \texttt{cs-503} allocation. Two agents are already training in parallel.
    \item \textbf{Software:} Habitat-sim 0.3.3, habitat-baselines (DD-PPO), PyTorch 2.1---all installed and tested. Custom foveated policy implemented and ready.
    \item \textbf{Risk mitigation:} The end-to-end foveated policy with learned gaze is the highest-risk component. If gaze learning collapses (e.g., converges to the image center), we fall back to fixed-center and random-gaze ablations, which still let us test H1 and H2. The blind and uniform agents---the lowest-risk part---are already training and on their own yield a Wijmans-style replication paired with a sighted comparison.
\end{itemize}
